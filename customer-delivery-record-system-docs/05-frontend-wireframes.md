# 前端页面草图说明

## 1. 说明

本文基于当前前端源码 `web/app.js` 与 `web/styles.css` 整理，描述当前后台页面布局、交互重点和样式策略，不是概念草图，而是“按当前界面实现”复原的文字版线框说明。

## 2. 整体视觉基调

### 2.1 色彩与氛围

1. 主色为深绿色系
2. 辅助色为灰蓝中性色
3. 页面背景使用浅灰底 + 绿色径向渐变
4. 头部为半透明白底 + 毛玻璃

### 2.2 组件风格

1. 卡片圆角较大
2. 表单控件统一圆角输入框
3. 列表和表格强调轻量线框
4. 主要按钮为绿色实底
5. 删除按钮为浅红底

### 2.3 内容密度策略

1. PC 后台优先
2. 列表内容通过 `ellipsis` 截断控制行高
3. 文档附件使用卡片网格，不直接铺开大图
4. 备注截图采用缩略图面板，避免一图占满一屏

3. 编辑态会展示已保存截图和待上传截图两块区域

## 16. 列表与长文本策略

1. 表格单元格统一使用截断容器，避免一行因长备注或长路径被撑高
2. SQL、配置内容等长文本不在列表直接展开，只在表单内查看或编辑
3. 文档资料页图片使用缩略图固定高度
4. 附件说明和备注只展示一小段摘要

## 17. 当前前端实现差异

1. `canEdit()` 当前对所有已登录用户返回 true，因此界面展示的编辑入口多于后端真实权限
2. 真正写权限以后端鉴权为准
3. 用户角色仍存在于后端，但未在当前界面暴露
